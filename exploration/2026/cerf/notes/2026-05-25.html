Summary

Major session covering codebase infrastructure, data source improvements, and new analysis notebooks. The main analytical work extended the trigger performance analysis to both Adamawa and Benue states via a new parameterised notebook (13_model_performance.ipynb). An initial version of 14_forecast_performance.ipynb was also created, introducing skill-vs-leadtime and exceedance detection analysis for both GloFAS and Google GRRR reforecasts.

Forecast performance — initial version (14_forecast_performance.ipynb)

Initial notebook created with the following structure:

  • Data loading: GloFAS reanalysis + ensemble reforecast; Google GRRR reanalysis + reforecast; both accessed via STATE_CONFIG
  • Return period thresholds: GloFAS thresholds derived from reanalysis annual wet-season maxima; Google thresholds from the official GCS return periods dataset
  • Skill vs leadtime: Spearman correlation and MAPE between daily reforecast and reanalysis values, restricted to wet-season days, grouped by leadtime — separately for GloFAS (ensemble mean) and Google
  • Annual exceedance detection: For each year and leadtime, whether the wet-season annual max ensemble-mean reforecast exceeded the RP threshold, compared against reanalysis — reporting hit rate and false alarm rate vs leadtime per RP level
